# 📱 手机使用指南 · 火山引擎售前学习 PWA

本应用已经做成 **PWA**（渐进式网页应用）。无论是鸿蒙、安卓还是 iPhone，都能"添加到主屏幕"作为图标启动，**离线可用**，体验跟普通 App 几乎一样。

下面三条路径**渐进可选**：先用 A 试一下没问题，再走 B 永久部署，C 是想要 APK 文件时再做。

---

## 🅰 路径 A · 局域网立即测试（5 分钟，无需注册任何账号）

适用：电脑和手机在**同一 WiFi**下，临时上手看效果。

### 步骤

1. **电脑保持开机 + 启动本地服务器**
   双击运行 `启动学习服务器.bat`（在项目根目录），看到 `Serving HTTP on 0.0.0.0 port 8766` 即成功。

2. **手机连同一 WiFi**

3. **手机浏览器打开**：
   ```
   http://192.168.1.6:8766/
   ```
   推荐用 **华为浏览器** 或 **Chrome**。

4. **添加到主屏幕**（让它像应用一样打开）：
   - 华为浏览器：右下"☰"菜单 → "添加到桌面"
   - Chrome：右上"⋮"菜单 → "添加到主屏幕"
   - 桌面会出现一个蓝紫渐变的"VE"图标，点击即打开

### ⚠️ 这条路径的限制

- **没有真正离线缓存**（局域网 HTTP 协议下 Service Worker 不工作；电脑关机或离开 WiFi 就访问不到）
- 适合先看看体验，**长期使用请走路径 B**

### Windows 防火墙挡住时

如果手机访问不通，电脑用**管理员 PowerShell** 跑一遍：
```powershell
New-NetFirewallRule -DisplayName "PWA Learn 8766" -Direction Inbound -LocalPort 8766 -Protocol TCP -Action Allow
```

## 🅱 路径 B · 永久部署到 GitHub Pages（推荐，30 分钟一次配好）

这是**让 PWA 真正变成"App"的正确做法**：HTTPS + 离线缓存 + 主屏幕图标 + 长按图标显示快捷方式。配好之后**电脑关机也能用、断网也能用**。

### 1. 注册 GitHub 账号（已有可跳过）
打开 https://github.com → Sign up

### 2. 创建仓库
- 点右上 "+" → "New repository"
- Repository name：`volcengine-presales-learn`（或任意英文名）
- Public（必须公开，免费版 Pages 才能用）
- 点 "Create repository"

### 3. 把项目代码推上去

如果你电脑装了 Git：
```bash
cd "E:/ai/claude/火山引擎学习"
git init
git add .
git commit -m "init pwa"
git branch -M main
git remote add origin https://github.com/你的用户名/volcengine-presales-learn.git
git push -u origin main
```

如果不熟悉 Git，最简单方式：
- 在 GitHub 仓库页面点 "uploading an existing file"
- 把 `火山引擎学习/` 文件夹**所有内容**（不含 `_work/` 和 `.claude/` 子目录）**拖进网页**
- 提交

### 4. 启用 GitHub Pages
- 仓库 → Settings → 左边栏 Pages
- Source 选 "Deploy from a branch"
- Branch 选 `main` + `/ (root)` → Save
- 等 1-2 分钟，页面顶部会显示绿色对勾 + 网址：
  ```
  https://你的用户名.github.io/volcengine-presales-learn/
  ```

### 5. 手机访问 + 安装

- 手机浏览器打开上面的 URL
- 浏览器底部/顶部会**自动弹出 "安装应用"** 提示，点同意即可
- 没自动弹？菜单里手动选 "添加到主屏幕" / "安装应用"
- 桌面出现"VE"图标 → 点开就是**全屏无浏览器条**的应用
- **长按图标**会看到 3 个快捷方式：仪表盘 / 竞品分析 / 面试培训（这是 manifest 里定义的）

### ✅ 路径 B 的好处
- **HTTPS 自动**（GitHub Pages 免费送 SSL 证书）
- **Service Worker 工作**（首次访问后离线也能学）
- **跨设备**（任何手机/电脑用同一 URL 都能访问）
- **更新简单**（你改代码 push 上去，10 分钟后所有手机自动拿到新版本）

## 🅲 路径 C · 转成真正的 APK 安装包（可选）

> ⚠️ **鸿蒙系统兼容性先确认一下**：
> - 如果你的手机是 **HarmonyOS 4.x / EMUI / 双框架版**（设置→关于手机能看到），**支持 APK 安装**，可以走这条路径。
> - 如果是 **HarmonyOS NEXT（纯鸿蒙内核）** —— 已**移除 AOSP 兼容层，不再支持 APK**。这种情况建议直接用路径 B（PWA 添加到主屏幕），效果其实和原生 App 一样。
> - 不确定是哪一种？打开"设置 → 关于手机 → 软件版本"看版本号；带 "HarmonyOS 4.x" 或 "EMUI" 字样的能装 APK，写着 "HarmonyOS NEXT" 或 "鸿蒙 5/6"（纯鸿蒙）的不能。

如果你希望在鸿蒙手机的"应用列表"里跟其他应用并列，而不是 PWA 图标，可以走这条路径。

> 工具：[**PWABuilder**](https://www.pwabuilder.com/) — 微软官方免费开源工具，**无需本地装 Android Studio**，全程在浏览器里完成。

### 步骤

1. **先做完路径 B**（拿到 GitHub Pages URL）

2. **打开 PWABuilder**：https://www.pwabuilder.com/

3. **粘贴你的 PWA 网址** → 点 "Start"
   - 它会自动检测你的 manifest 和 service worker
   - 三项都打勾才能继续（Manifest ✓ Service Worker ✓ HTTPS ✓）

4. **点 "Package For Stores" → 选 "Android"**

5. **选 Build Type**：
   - 推荐 **"Recommended"** （默认 TWA · Trusted Web Activity）
   - 包内置 Chrome 内核运行你的 PWA，**鸿蒙完美兼容**

6. **填一些信息**：
   - Package ID：例如 `com.volcengine.presales.learn`
   - App name：火山售前学习
   - Version：1.0.0
   - 其他保持默认即可

7. **下载 .zip**，里面包含：
   - `app-release-signed.apk` ← **直接安装到手机**
   - `app-release-bundle.aab` ← 上架 Google Play 用，鸿蒙暂时用不到

8. **传到鸿蒙手机安装**：
   - 把 .apk 通过微信/QQ/数据线发到手机
   - 鸿蒙 6.1：设置 → 安全和隐私 → 更多安全设置 → **允许从未知来源安装应用**（针对文件管理器/微信开启）
   - 用文件管理器打开 .apk → 安装
   - 桌面出现独立的"火山售前学习"应用图标，跟正常 App 一样

### ⚠️ APK 安装注意

- 鸿蒙 6.1 对未签名/外部 APK 会提示"风险应用"，是正常现象
- TWA 模式的 APK **本质仍是从 GitHub Pages 加载**，所以 GitHub Pages 不能停（PWABuilder 会用 Digital Asset Links 验证 URL 归属）
- **第一次启动**仍需联网（拉缓存），之后离线可用

## 🅳 路径 D · 纯本地离线分发（不上网，进阶）

适合不方便上 GitHub 的场景。

### 方法：在手机上跑一个轻量 HTTP server

1. 鸿蒙手机装 **Termux**（应用市场搜不到时用 F-Droid / GitHub release）
2. 把整个 `火山引擎学习/` 文件夹复制到手机
3. Termux 里：
   ```bash
   pkg install python
   cd /sdcard/火山引擎学习
   python -m http.server 8766
   ```
4. 手机浏览器打开 `http://localhost:8766/`
5. localhost 算"安全上下文"，**Service Worker 能正常工作**，离线 OK

> 这条路径稍折腾。除非你有特殊需求（比如长期没网络、不想上传到 GitHub），**优先走路径 B**。

## 🆘 常见问题

**Q：我打开网页没看到"添加到主屏幕"提示？**
A：① 确认 URL 是 HTTPS（路径 B/C） ② 在浏览器菜单里手动选"添加到主屏幕"。鸿蒙浏览器可能在"更多" → "添加到主屏"。

**Q：图标安装后是空白/默认网页缩略图？**
A：清下浏览器缓存重试，或确认 `icons/icon-512.png` 已部署成功。

**Q：PWA 启动后是网页样子，不是全屏 App？**
A：检查 `manifest.webmanifest` 里 `"display": "standalone"` 是否生效；某些老浏览器仅 `display: minimal-ui` 起作用。

**Q：每次更新代码后，手机端怎么拿到新版本？**
A：① GitHub push 后 GitHub Pages 自动重建（约 1 分钟） ② 手机 PWA 启动时 SW 后台检查更新 ③ **完全关闭并重启 App**即可加载新版（已写在 `[PWA] 已下载新版本，下次启动应用` 提示里）。
- 想强制刷新：浏览器里访问 URL → 长按刷新按钮（Chrome 是 Ctrl+F5 / 移动端清除站点数据）

**Q：能不能装上 Google Play？**
A：可以，路径 C 生成的 .aab 上传 Play Console 即可。但鸿蒙不依赖 Play，APK 本地安装就够了。

## 📝 学习数据保存在哪里？

所有进度（已完成天数、章节小测分数、考试历史）都存在**手机浏览器/PWA 的 localStorage** 里。

- 不会上传到任何服务器
- 同一手机的同一浏览器会保留
- **换手机/换浏览器需要重新开始**（这是纯前端应用的天然限制）
- 如果你需要跨设备同步，告诉我，可以加 GitHub Gist / Supabase 做云同步（属于扩展功能）
