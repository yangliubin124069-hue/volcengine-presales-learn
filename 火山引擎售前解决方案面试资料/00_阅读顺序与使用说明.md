## 二、面试准备核心策略

火山引擎售前解决方案岗位的关键不是“背产品参数”，而是证明你能完成这条链路：

客户业务目标 -> 痛点和约束 -> 技术架构 -> 产品组合 -> POC 验证 -> 商务价值 -> 交付边界 -> 风险闭环

回答时建议使用这个结构：

1. 先复述客户目标，确认问题边界。
2. 拆成业务指标、技术指标、合规指标、成本指标。
3. 给出一版分层架构，说明每层选择理由。
4. 说明 POC 怎么验，成功标准是什么。
5. 主动补充风险、替代方案和后续推进计划。

## 三、你需要准备的个人素材

请提前准备 3 个自己的 STAR 案例：

1. 技术方案案例：你如何把业务需求转成架构方案。
2. 客户/跨团队推进案例：你如何处理分歧、压缩周期、推动落地。
3. 问题解决案例：POC、交付、性能、稳定性或成本问题如何定位解决。

每个案例按这个模板整理：

背景：客户是谁，业务目标是什么。  
任务：你负责什么，难点是什么。  
行动：你做了哪些调研、设计、协调和验证。  
结果：指标提升、成本下降、效率提升、项目推进结果。  
复盘：如果重做，你会怎么改进。
